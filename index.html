Скрин-ридеры игнорируют

  • img с пустыми alt аттрибутами alt=""
  • display: none;
  • visibility: hidden;
  • :before и :after content (Скрин-ридеры читают не все символы)

Скрин-ридеры видят

  • Контент, спрятанный через opacity, z-index, height
  • oﬀ-screen позиционирование (text-indent, top, left)
  • CSS clipping

WAI-ARIA

Web Accessibility Initiative
Accessible Rich Internet Applications

WAI-ARIA

Cтандарт доступности активных
веб-приложений

WAI-ARIA предоставляет нам

  • Роли для описания типа активных элементов
  • Роли для описания структуры страниц
  • Атрибуты для описания состояний
  • Атрибуты для задания активных областей страницы, допускающих обновления
  • Атрибуты для перетаскивания объектов

Плюсы

  • Мы обеспечиваем аддекватную семантическую информацию
    и лучшее взаимодействие со скрин-ридерами
  • Пишем прямо в HTML
  • WAI-ARIA это часть спецификации HTML
  • Легко манипулировать с помощью JS
  • И это всё не заметно для обычных пользователей

Роли

  • Мы можем создать новый семантический смысл элементов используя “role-” аттрибуты
  • Принцип "Установи и забудь"
  • <li role="list">
  • <main role="main">
  • <table role="presentation">

Используйте ARIA аккуратно

  • ARIA это улучшение, а не замена.
  • Это не магия и не панацея от всех бед
  • Не изобретайте велосипеды, сначала посмотрите спецификацию
