Screen Readers - польза использования

  • Используйте клавиатуру для навигации и поиска поконтенту
  • 98.6% пользователей screen reader'ов users не выключают JS!
  • Приличные возможности по кастомизации

HTML Считается

Headings

  • Основная способ навигации пользователей screen reader'ов
  • Do your main content areas have headings?
  • Are they descriptive?
  • Они следуют иерархии? (h1 >> h6)

Screen Reader игнорирует

  • img с пустыми alt аттрибутами alt=""
  • display: none;
  • visibility: hidden;
  • :before content, :after content (тип того)
  • Дуржите в голове иконки и иконочные шрифты!

Screen Reader не забудет

  • Контент, спрятанный через opacity, z-index, height
  • oﬀ-screen позиционирование (text-indent, top, left)
  • CSS clipping
			.wcag-hidden {
				border: 0;
				clip: rect(0 0 0 0);
				height: 1px;
				margin: -1px;
				padding: 0;
				position: absolute;
				width: 1px;
			}

За границами банального

Должно же быть что-то еще, да?

WAI-ARIA

  • Семантическая информация и лучшее взаимодействие со Screen Reader'ами
  • Пишем прямо в HTML
  • Часть спецификации HTML5
  • Roles, states & properties
  • Не аффектает стили "нормального" поведения

Roles

  • Создать новый семантический смысл элементов используя “role-” аттрибуты
  • Once set, they don’t change
  • <nav role="navigation">
  • <article role="article">
  • <div role="tablist">
  • <div role="combobox>

Widget Roles

  • Семантический смысл для ваших кастомных компонентов
  • tooltip
  • slider
  • dialog
  • tab
  • menu
  • alert
  • and много чего еще!

States & Properties

  • Описывают отношения - между контентом и взаимодеййствием с пользователем
  • Updated via JS on UI changes
  • aria-аттрибуты
  • <section aria-labelledby="MainHeading">
  • <div aria-expanded="true">
  • и т.д.

Content Relationships

  • Семантически залинкуйте label наконтент или добавьте, если их не было
  • aria-labelledby, aria-label

Используйте ARIA аккуратно

  • ARIA это связь, а не замена.
  • Используйте простой HTML так, где это возможно
  • Это не магия и не панацея
  • Ответственность за events, focus management, keyboard support и смысл всё еще лежит на вас
  • Есть только один способ быть на 10% уеренным... Потестить
